# 后续任务清单

最后更新：2026-05-26

## 执行原则

- 先做 P0，再做 P1，最后才考虑 P2。
- 每次只拿一个明确任务做，不要一次性重构整个项目。
- 任何代码修改完成后，至少运行 `npm.cmd test` 和 `npm.cmd run build`。
- 每次任务完成后立即更新 `docs/context.md`；如果任务状态变化，同步更新本文件。
- 后续科研、设计、代码和数据处理任务完成后，同步维护 `docs/design_log.md`，必要时新增 `docs/design_reports/` 下的阶段性设计报告。
- 当前优先级不是继续美化界面，而是让项目进入可长期维护、可接入实验数据的状态。

## P0：必须马上做

- 确认源码中文编码问题：用 VS Code 和浏览器实际显示核对 `index.html`、`src/app.js`、`src/main.js`、`src/app.test.js` 中的中文文案；如果只是 PowerShell 显示乱码，不要修改；如果源文件确实损坏，再小范围修复。

- 确认采集与控制硬件链路：核对“汇川 AI820”的准确型号、输入类型、采样频率、通信协议，以及三轴振动和声发射是否需要独立高速采集卡。
- 明确第一版粗糙度预测演示目标：只展示预测结果，还是需要能读取一段实验数据并输出 Ra。
- 结合师兄视频要求，继续明确后续真实仿真边界：当前已完成第一版 Abaqus 简化移动热源温度场链路，后续需要确认真实热源功率、材料热参数、接触/摩擦热比例、散热边界和验证方式。
