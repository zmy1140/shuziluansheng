# 下一步任务清单

最后更新：2026-05-18

本清单面向科研推进和设计报告沉淀，重点说明“接下来为什么做、做到什么程度、需要什么依据”。工程实现层面的长期待办仍以 `docs/todo.md` 为准。

## P0：立即推进

1. 确认中文编码与浏览器显示
   - 目标：判断当前 `src/app.js`、`src/main.js`、`src/app.test.js` 中中文是否真实损坏。
   - 验收：明确“只是 PowerShell 显示乱码”或“文件内容确实损坏”；如确实损坏，再小范围修复。

2. 完成 3D 加工动画最小演示
   - 目标：在不破坏 GLB 导入能力的前提下，实现简单工件、磨头/末端工具、预设轨迹和局部颜色渲染。
   - 边界：第一版只允许标注为“模拟加工演示 / 数据映射预留”，不得写成真实仿真或真实加工过程。

3. 定义第一版实验数据字段表
   - 目标：形成时间戳、六维力、三轴加速度、声发射、主轴状态、粗糙度 Ra 标签等字段。
   - 需确认：采样频率、单位、窗口长度、同步方式、标签来源。

4. 确认采集链路和硬件型号
   - 目标：区分高频振动/声发射采集设备与 PLC 低频状态采集的职责。
   - 需确认：“汇川 AI820”的准确型号和规格，YE6275D 与三轴加速度传感器的实际接线和采样配置。

## P1：近期推进

1. 建立本地 CSV/JSON 回放样例
   - 目标：用结构化文件替代纯随机模拟数据。
   - 注意：若使用伪真实数据，必须在界面和报告中明确标注。

2. 设计 Abaqus 结果到前端的映射格式
   - 目标：明确 ODB 后处理输出 JSON/CSV 后，如何映射到前端曲线、局部颜色或工件区域。
   - 需确认：真实仿真输出变量，例如位移、应力、接触压力、温度或磨抛影响指标。

3. 整理第一阶段演示说明
   - 目标：形成可给师兄/导师汇报的说明材料。
   - 内容：当前已完成、当前均为模拟数据、后续如何接真实实验/仿真/预测模型。

4. 持续更新设计报告
   - 目标：每次完成重要设计、仿真、实验、代码或数据处理任务后，同步写入 `docs/design_log.md`，必要时新增阶段性设计报告。
