# 下一步任务清单

最后更新：2026-05-26

本清单面向科研推进和设计报告沉淀，重点说明“接下来为什么做、做到什么程度、需要什么依据”。工程实现层面的长期待办仍以 `docs/todo.md` 为准。

## P0：立即推进

1. 升级第一版简化热源仿真链路
   - 当前状态：已完成 `100 mm x 100 mm x 8 mm` 平板 + `24 mm` 等效磨头路径的第一版 Abaqus 简化移动热源链路，并已导出 `x_mm,z_mm,temperature_c` CSV 替换前端温度场 JSON。
   - 下一目标：把当前占位“移动接触温度边界”升级为有依据的热流功率、摩擦生热比例或热-力接触模型。
   - 需要补充：材料热导率、比热、密度、环境换热、磨头转速/法向力/摩擦系数、接触面积与热分配比例。
   - 验收：前端仍读取同一 CSV/JSON 格式，但文档能说明热源参数来源、边界条件来源和结果适用范围。
   - 边界：在参数未经实验或文献验证前，仍只能写成“简化热源仿真链路验证”，不能写成真实加工温度结论。

2. 确认采集链路和硬件型号
   - 目标：区分高频振动/声发射采集设备与 PLC 低频状态采集的职责。
   - 当前依据：`AI820` 已按汇川 PAC 智能控制器记录，不再按普通模拟量输入模块理解；`YE6275D` 可作为三轴加速度候选采集器。
   - 需确认：M3553C 标定参数、M8229 实际通信方式、YE6275D 实际接线和采样配置、声发射设备资料、主轴 Modbus 寄存器地址。

3. 验证第一版实验数据字段表
   - 状态：已新增 `docs/实验数据字段表.md`，当前为采集归档字段草案，没有真实采集数据。
   - 目标：后续拿真实采集文件或设备导出样例核对字段名、单位、采样率、通道映射、时间同步和 Ra 标签对应关系。

4. 确认中文编码与浏览器显示
   - 目标：判断当前 `src/app.js`、`src/main.js`、`src/app.test.js` 中中文是否真实损坏。
   - 验收：明确“只是 PowerShell 显示乱码”或“文件内容确实损坏”；如确实损坏，再小范围修复。

## P1：近期推进

1. 建立监测信号本地 CSV/JSON 回放样例
   - 状态：路径 CSV 到路径 JSON、温度场 CSV 到温度场 JSON 已完成第一版；力、振动、声发射、主轴和 Ra 的回放尚未完成。
   - 目标：在已完成路径和温度场格式的基础上，继续用结构化文件替代纯随机模拟数据。
   - 前提：优先基于 `docs/实验数据字段表.md` 选择窗口级摘要字段，不直接把高频原始波形塞进前端大 JSON。
   - 注意：若使用伪真实数据，必须在界面和报告中明确标注。

2. 完善 Abaqus 结果到前端的映射格式
   - 状态：已完成简化厚板 Mises 应力映射原型、温度场 CSV/JSON 第一版格式，以及移动热源温度场 ODB 后处理脚本。
   - 下一步：继续记录不同 Abaqus 输出变量、不同曲面/网格形式如何归一到 `x_mm,z_mm,temperature_c` 或后续扩展字段。
   - 仍需确认：材料热参数、热源模型、接触/摩擦热比例、散热边界、单位、参考温度、最大/最小值和色带范围。

3. 确认真实仿真和实验边界
   - 目标：把“当前简化厚板映射原型”和“未来真实磨抛仿真”的边界写清楚。
   - 需确认：真实工件几何、涂层材料、磨头接触形式、载荷/位移边界、轨迹输入、输出变量和验证数据。

4. 整理第一阶段与第二阶段演示说明
   - 目标：形成可给师兄/导师汇报的说明材料。
   - 内容：当前已完成的前端壳体、3D 加工动画、Abaqus 映射原型，当前均为模拟或链路验证数据，后续如何接真实实验/仿真/预测模型。

5. 持续更新设计报告
   - 目标：每次完成重要设计、仿真、实验、代码或数据处理任务后，同步写入 `docs/design_log.md`，必要时新增阶段性设计报告。
